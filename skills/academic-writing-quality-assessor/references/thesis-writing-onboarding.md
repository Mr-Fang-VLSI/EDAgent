# Thesis Writing Onboarding

Use this note when the active task is dissertation assembly rather than experimentation.

## Target task shape

Typical input set:
- one thesis/dissertation format example,
- one proposal or partial thesis draft,
- one or more published/submitted papers,
- one active Overleaf or local LaTeX project.

## Recommended execution order

1. pick the active dissertation skeleton
2. identify which chapters are truly active in `main.tex`
3. merge paper content into thesis chapters instead of preserving conference-paper structure
4. normalize front matter before deep content editing
5. clean stale chapters and duplicate figure variants after local backup
6. run terminology/grammar QA only after chapter structure stabilizes

## Minimal-intrusion principle

Prefer:
- correcting chapter mapping,
- unifying naming and paths,
- fixing front matter,
- removing obsolete material from cloud projects after local archive,
- making localized wording fixes.

Avoid:
- rewriting full chapters before structure is stable,
- leaving parallel old/new chapter directories active,
- mixing proposal language into dissertation front matter.

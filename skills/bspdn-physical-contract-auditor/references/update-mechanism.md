# Update Mechanism

Refresh this skill's summarized knowledge when any of the following happens:

## Refresh triggers

1. a new local paper summary materially changes the expected `nTSV/BPR/BM1/BM2/M0/M1` narrative
2. GT3 or proxy techlef collateral changes layer names, indices, or via rules
3. a micro-test directly confirms or contradicts a current working hypothesis
4. `gt3-backside-route-policy` changes its default physical contract

## What to update

1. refresh `background-knowledge-links.md`
2. update any Gate A assumptions in `docs/knowledge_base/107_BSPDN_TOPOLOGY_AND_BENEFIT_VALIDATION_PROGRAM_20260307.md`
3. if the change is durable, append a reusable lesson through `eda-experiment-phenomenology-analyst`

## Promotion rule

Promote a hypothesis into stable background knowledge only when:
1. local collateral evidence and
2. run-time or micro-test evidence

both support the same interpretation strongly enough to stop calling it a mere working assumption.

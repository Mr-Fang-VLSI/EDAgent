# Visual Polish & Final Review Checklist

This checklist is for **Phase 4** of the academic presentation workflow. It ensures the final slide deck is visually coherent, readable, and professional.

## 1. Readability & Typography

- [ ] **Minimum Font Size**: Is all text at least **18pt**? This includes text inside diagrams, charts, and tables. This is a critical accessibility and readability standard for conference presentations.
- [ ] **Font Consistency**: Is the same font family used throughout the presentation for titles, subtitles, and body text? Refer to the `style_instruction` from `slide_initialize`.
- [ ] **Hierarchy**: Is there a clear visual hierarchy? Titles should be significantly larger than subtitles, which should be larger than body text.

## 2. Layout & Consistency

- [ ] **Layout Consistency**: Is a consistent layout applied across all content slides? For academic topics, a **left-aligned, single-column format** is generally preferred for clarity. Avoid mixing centered, left-aligned, and multi-column layouts without a strong reason.
- [ ] **White Space**: Is there sufficient white space on each slide? Avoid cluttering slides with too much text or too many images. A clean layout is easier to read and understand.
- [ ] **Alignment**: Are all elements (text boxes, images, charts) properly aligned with each other? Use alignment tools to ensure a clean, organized look.

## 3. Visual Cohesion & Aesthetics

- [ ] **Color Palette**: Are the colors used consistent with the `style_instruction` color palette? The color scheme should be uniform across all slides.
- [ ] **Image Quality**: Are all images high-resolution and free of watermarks? Low-quality or watermarked images detract from the professionalism of the presentation.
- [ ] **Visual Style**: Do all visual elements, including charts and diagrams, share a consistent style? For example, they should use the same color palette and font styles.

## 4. Final Review & Delivery

- [ ] **Page Numbering**: Are all slides correctly numbered?
- [ ] **Typos & Grammar**: Has the entire presentation been proofread for spelling and grammatical errors?
- [ ] **Presentation Run-Through**: Use the `slide_present` tool to conduct a final, full review of the presentation. Click through every slide to catch any rendering issues, formatting errors, or content mistakes.
- [ ] **File Integrity**: Before delivering, ensure the project directory is complete and all assets are correctly linked.

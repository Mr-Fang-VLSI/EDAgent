# Naming Normalization

Rename artifacts when:
1. the current name is misleading,
2. the name no longer reflects the file's role,
3. inconsistent naming makes lookup difficult.

Preferred naming qualities:
1. scope is obvious,
2. purpose is obvious,
3. version or date is included only when operationally necessary,
4. names are consistent with neighboring canonical artifacts.

When renaming:
1. update references if they are maintained source files,
2. record old-to-new mapping,
3. avoid cosmetic churn that does not improve clarity.

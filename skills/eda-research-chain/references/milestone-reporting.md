# Research Chain Milestone Reporting

## When To Report

Do not generate milestone slides or PDF summaries for every process stage by default.

Generate a milestone report only when a new version achieves a validated improvement threshold, for example:
- first reach `>=2%` power reduction with non-worse area/timing,
- or a later milestone that is explicitly stronger than the last validated version.

## Required Content

Each milestone report should include:
1. method delta,
2. key conclusion,
3. evidence table,
4. next-step plan.

## Guardrail

If validation artifacts are incomplete or the claimed improvement is not yet gated, do not generate milestone reporting artifacts.

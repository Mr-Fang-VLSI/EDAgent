# Infrastructure Reporting

Use this reference when writing the closeout for an infrastructure maintenance task.

## Required Report Content

1. changed files
2. guard/audit artifact paths
3. reference-topology decision for each touched skill
4. utility-vs-execution-vs-theory boundary decision if the task reorganized skills
5. residual risk
6. rollback trigger

## Utility Reorganization Note

If the task touched utility skills, the report should also state:
1. which horizontal capability owner was strengthened or introduced,
2. which downstream skills should now delegate instead of duplicating logic,
3. whether any remaining duplication still exists.

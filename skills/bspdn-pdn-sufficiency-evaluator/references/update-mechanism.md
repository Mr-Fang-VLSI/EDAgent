# Update Mechanism

Refresh this skill's summarized knowledge when:

## Refresh triggers

1. Gate C experiments repeatedly show a stable new pattern across cases
2. the physical contract changes after a topology audit
3. the active GT3 PDN settings change materially
4. new local paper evidence changes the interpretation of BPR-centered PDN design

## What to update

1. refresh `background-knowledge-links.md`
2. update Gate C assumptions inside `docs/knowledge_base/107_BSPDN_TOPOLOGY_AND_BENEFIT_VALIDATION_PROGRAM_20260307.md`
3. if a cross-batch lesson becomes reusable, write it into the experience layer through `eda-experiment-phenomenology-analyst`

## Stability rule

Do not change the evaluator's default contract from one-off experimental noise.
Require repeated or unusually decisive evidence before revising the sufficiency interpretation.

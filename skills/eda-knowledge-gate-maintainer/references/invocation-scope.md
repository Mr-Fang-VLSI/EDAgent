# Invocation Scope

Use this reference when deciding how `eda-knowledge-gate-maintainer` should be invoked.

## Preferred Path

1. Prefer invocation through `workflow-scoped-execution` for routine scoped execution.
2. Use direct invocation only for gate-hygiene repair, debugging, or policy validation.

## Scope Requirement

The call should provide:
1. an interaction scope name,
2. a task brief path,
3. enough keywords to run a meaningful tool query.
